# AI Context - Teacher Corner 2.0

> This file provides AI coding assistants with essential context about the Teacher Corner 2.0 project.

## Project Summary

**Teacher Corner 2.0** is an enterprise-grade educational platform built with Angular 14, Firebase, and GraphQL. It supports multiple client deployments (staging, Jigyaasa) and runs as both a web application and desktop app via Electron.

**Tech Stack:**
- Angular 14.1.0 + TypeScript 4.7.4
- Firebase (Firestore, Auth, Storage, Analytics)
- Apollo Client + GraphQL
- Angular Material + Tailwind CSS + Fuse Design System
- Electron 37.2.0 (Desktop)

## Architecture Overview

### Frontend Architecture
- **Framework**: Angular 14 with modular architecture
- **Routing**: Angular Router with lazy loading
- **State**: Apollo Client cache + Firebase real-time observables
- **UI**: Angular Material + Tailwind utilities + Fuse components
- **Forms**: Reactive Forms with Material form fields

### Backend Integration
- **Database**: Firebase Firestore (NoSQL document database)
- **Authentication**: Firebase Auth (email/password, OAuth)
- **Storage**: Firebase Cloud Storage (file uploads)
- **Analytics**: Firebase Analytics + Google Analytics 4
- **API**: GraphQL via Apollo Client

### Design System
- **Fuse Framework**: Custom design system (src/app/@fuse/)
  - Navigation components (horizontal, vertical)
  - Multiple layout variants
  - Theme system with Material integration
  - Reusable UI components

## Code Organization Principles

### Module Strategy
1. **Core Module** (singleton): Services, guards, interceptors loaded once
2. **Shared Module**: Reusable components, pipes, directives
3. **Feature Modules**: Domain-specific functionality (lazy loaded where appropriate)
4. **Fuse Module**: Design system components

### Service Layer
- **Firestore Services**: Database operations (database-module/firestore/)
  - Pattern: `[Entity]FirestoreService`
  - Methods: `get()`, `create()`, `update()`, `delete()`
  - Returns: RxJS Observables

- **Business Logic Services**: Feature-specific logic (src/app/core/services/)
  - Pattern: `[Feature]Service`
  - Provided in root or feature module

- **GraphQL Services**: API calls via Apollo Client

### Component Patterns
- **Smart/Container Components**: Handle data, business logic
- **Dumb/Presentational Components**: Pure UI, inputs/outputs only
- **OnPush Change Detection**: Used for performance-critical components

#### Component Structure
```typescript
@Component({
  selector: 'app-quiz-list',
  templateUrl: './quiz-list.component.html',
  styleUrls: ['./quiz-list.component.scss'],
  changeDetection: ChangeDetectionStrategy.OnPush
})
export class QuizListComponent implements OnInit, OnDestroy {
  // Public properties
  quizzes$: Observable<Quiz[]>;

  // Private properties
  private _unsubscribeAll: Subject<void> = new Subject();

  // Constructor with DI
  constructor(
    private quizService: QuizService,
    private router: Router
  ) {}

  // Lifecycle hooks
  ngOnInit(): void {
    this.loadQuizzes();
  }

  ngOnDestroy(): void {
    this._unsubscribeAll.next();
    this._unsubscribeAll.complete();
  }

  // Public methods
  loadQuizzes(): void {
    this.quizzes$ = this.quizService.getAll()
      .pipe(takeUntil(this._unsubscribeAll));
  }
}
```

#### Service Structure
```typescript
@Injectable({
  providedIn: 'root'
})
export class QuizService {
  constructor(
    private firestore: AngularFirestore,
    private apollo: Apollo
  ) {}

  getAll(): Observable<Quiz[]> {
    return this.firestore
      .collection<Quiz>('quizzes')
      .valueChanges();
  }

  getById(id: string): Observable<Quiz> {
    return this.firestore
      .doc<Quiz>(`quizzes/${id}`)
      .valueChanges();
  }

  create(quiz: Quiz): Promise<void> {
    const id = this.firestore.createId();
    return this.firestore
      .doc(`quizzes/${id}`)
      .set({ ...quiz, id });
  }

  update(id: string, changes: Partial<Quiz>): Promise<void> {
    return this.firestore
      .doc(`quizzes/${id}`)
      .update(changes);
  }

  delete(id: string): Promise<void> {
    return this.firestore
      .doc(`quizzes/${id}`)
      .delete();
  }
}
```

  private _unsubscribeAll = new Subject<void>();

  ngOnInit(): void {
    this.dataService.getData()

  }

  ngOnDestroy(): void {
    this._unsubscribeAll.next();

  }
}

## Performance Best Practices

1. **Change Detection**: Use `ChangeDetectionStrategy.OnPush` for components
2. **Lazy Loading**: Feature modules loaded on-demand
3. **TrackBy Functions**: Always use with `*ngFor`
   ```typescript
   trackByFn(index: number, item: Quiz): string {
     return item.id;
   }
   ```
4. **Pure Pipes**: Mark pipes as pure unless necessary
5. **Virtual Scrolling**: Use CDK virtual scroll for long lists
6. **Unsubscribe**: Always clean up subscriptions in `ngOnDestroy`
7. **OnPush + Async Pipe**: Optimal combination for reactive data

## Security Considerations

1. **Firebase Rules**: Configured in Firebase Console
2. **Auth Guards**: Protect routes requiring authentication
3. **Role-Based Access**: Check user roles before sensitive operations
4. **Input Sanitization**: Use Angular's built-in sanitization
5. **Encryption**: CryptoJS for client-side encryption (limited use)

1. **Angular 14**: Not the latest Angular version (current is v17+)

3. **Memory**: Large builds require `--max_old_space_size=8192`
4. **Multi-Tenancy**: Separate Firebase projects per deployment
5. **Offline Support**: Partial support via Electron offline mode

## Troubleshooting

### Build Errors
- **Memory issues**: Increase Node memory (`--max_old_space_size`)
- **TypeScript errors**: Check `tsconfig.json` settings
- **Module not found**: Verify import paths

### Firebase Errors
- **Permission denied**: Check Firestore rules
- **Auth errors**: Verify Firebase config in environment files
- **Quota exceeded**: Check Firebase usage limits

### Styling Issues
- **Tailwind not working**: Run `npm run build` to rebuild
- **Material theme**: Check theme imports in `styles.scss`
