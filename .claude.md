# Teacher Corner 2.0 - Claude Context

> This file provides Claude Code with essential context about the Teacher Corner 2.0 project architecture, conventions, and workflows.

## Project Overview

**Teacher Corner 2.0** is an Angular 14 educational platform built with:
- **Frontend**: Angular 14.1.0 with Material Design + Tailwind CSS
- **Backend**: Firebase (Firestore, Storage, Auth, Analytics)
- **State Management**: Apollo Client + GraphQL
- **Desktop Support**: Electron 37.2.0
- **Design System**: Custom Fuse design framework

This is a multi-tenant education platform with separate deployments for staging, and Jigyaasa environments.

## Key Architectural Patterns

### 1. Module Organization
- **Feature modules** follow Angular best practices (lazy loading where appropriate)
- **Core module** contains singleton services, HTTP interceptors, and guards
- **Shared module** exports commonly used components, directives, and pipes
- **Fuse module** provides design system components and layouts

### 2. Firebase Integration
- Located in `database-module/`
- Service classes follow pattern: `[Entity]FirestoreService`
- Uses AngularFire compatibility mode (`@angular/fire/compat`)
- All Firestore operations use observables (RxJS)

Example:
```typescript
// database-module/firestore/quiz.firestore.service.ts
export class QuizFirestoreService {
  collection = this.firestore.collection('quizzes');

  getQuiz(id: string): Observable<Quiz> {
    return this.collection.doc(id).valueChanges();
  }
}
```

### 3. GraphQL Integration
- Apollo Client configured in `graphql-module/`
- GraphQL queries/mutations in `*.graphql.ts` files
- Uses `apollo-angular` for Angular integration

### 4. State Management
- Combination of Apollo Client cache (GraphQL) + Firebase real-time observers
- Component state using RxJS BehaviorSubjects
- No dedicated state management library (NgRx, Akita, etc.)

### 5. Routing & Guards
- Route guards in `src/app/core/guards/`
- Firebase Auth Guards for protected routes
- Lazy-loaded feature modules

## Coding Conventions

### TypeScript Style
- **Interfaces**: PascalCase, prefix with `I` optional (e.g., `Quiz`, `IUser`)
- **Classes**: PascalCase (e.g., `QuizService`, `DashboardComponent`)
- **Constants**: UPPER_SNAKE_CASE (e.g., `MAX_ATTEMPTS`, `API_ENDPOINT`)
- **Variables/Functions**: camelCase (e.g., `getUserData()`, `currentUser`)
- **Private members**: prefix with `_` (e.g., `private _destroy$`)

### Angular Conventions
- **Components**:
  - Suffix with `Component` (e.g., `QuizListComponent`)
  - Selector prefix: `app-` (e.g., `app-quiz-list`)
  - Use OnPush change detection where possible

- **Services**:
  - Suffix with `Service` (e.g., `AuthService`)
  - Provide in root unless feature-specific
  - Use dependency injection

- **Pipes**:
  - Suffix with `Pipe` (e.g., `SafeHtmlPipe`)
  - Mark as `pure: true` unless necessary

### File Naming
- Components: `component-name.component.ts`
- Services: `service-name.service.ts`
- Guards: `guard-name.guard.ts`
- Pipes: `pipe-name.pipe.ts`
- Modules: `module-name.module.ts`

### SCSS/CSS Conventions
- **Tailwind utilities** preferred for spacing, colors, flexbox
- **SCSS** for complex component-specific styles
- **Fuse theme mixins** available for theming
- Use Material theming system for Angular Material components

Example:
```html
<!-- Prefer this -->
<div class="flex items-center gap-4 p-6 bg-card rounded-lg">

<!-- Over this -->
<div class="custom-container"></div>
```

## Common Patterns

### 1. Component Lifecycle with Cleanup
```typescript
export class MyComponent implements OnInit, OnDestroy {
  private _unsubscribeAll: Subject<void> = new Subject<void>();

  ngOnInit(): void {
    this.dataService.getData()
      .pipe(takeUntil(this._unsubscribeAll))
      .subscribe(data => {
        // handle data
      });
  }

  ngOnDestroy(): void {
    this._unsubscribeAll.next();
    this._unsubscribeAll.complete();
  }
}
```

### 2. Firestore CRUD Operations
```typescript
// Create
this.firestoreService.create(data);

// Read
this.firestoreService.get(id).subscribe();

// Update
this.firestoreService.update(id, changes);

// Delete
this.firestoreService.delete(id);
```

### 3. Form Handling
- Use Reactive Forms (`FormBuilder`, `FormGroup`, `FormControl`)
- Validators defined in form initialization
- Material Form Field components

### 4. Error Handling
- HTTP errors caught in services
- User-friendly error messages
- Console logging for debugging

## Performance Considerations

### Optimization Patterns
- **OnPush change detection** for performance-critical components
- **Lazy loading** for feature modules
- **Virtual scrolling** for long lists (Angular CDK)
- **trackBy** functions for `*ngFor` loops
- **Pure pipes** for transformations
- **Service Worker** for PWA capabilities (configured in `ngsw-config.json`)

### Memory Management
- Always unsubscribe from observables (use `takeUntil` pattern)
- Avoid memory leaks in event listeners
- Dispose of large objects in `ngOnDestroy`

## Known Issues & Gotchas

1. **Angular 14** - Be aware this is not the latest Angular version
2. **Firebase compat mode** - Uses compatibility API, not modular SDK
3. **Memory limits** - Build scripts use `--max_old_space_size=8192` for large builds
4. **Electron offline mode** - Special handling for offline desktop features
5. **Multi-tenancy** - Different Firebase projects per client
